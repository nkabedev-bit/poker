# План работ: Синхронизация времени таймера (Clock Drift Fix)

## Проблема
Клиентский таймер рассчитывает оставшееся время раунда в реальном времени, сравнивая локальное время устройства (`new Date()`) с серверной меткой начала уровня (`levelStartedAt`). Разница в системном времени между сервером базы данных и клиентским устройством (ТВ, планшетом) приводит к скачкам времени, зависаниям или преждевременному переключению блайндов.

---

## Предлагаемое решение
Использовать клиент-серверное смещение времени (Offset). При загрузке страницы вычисляется разница между серверным временем и локальным временем устройства, которая затем применяется ко всем расчетам.

---

## Пошаговый план реализации

### Шаг 1. Создание механизма расчета смещения на клиенте
В `components/public/public-screen.tsx` и `components/admin/timer-controls.tsx`:
1. Найти место инициализации состояния часов (обычно `useState(() => new Date(serverNowIso))`).
2. Сохранить вычисленное смещение времени в `useRef` или `state`:
   ```typescript
   const clientNow = Date.now();
   const serverTime = new Date(serverNowIso).getTime();
   const clockOffset = serverTime - clientNow; // смещение в миллисекундах
   ```
3. При каждом последующем обновлении состояния через Supabase Realtime (если приходит обновленное серверное время) актуализировать `clockOffset`.

### Шаг 2. Замена вызовов `new Date()` на скорректированное время
1. В `useEffect` при тике интервала вызывать функцию корректировки:
   ```typescript
   const getAdjustedDate = () => new Date(Date.now() + clockOffset);
   ```
2. Изменить интервальный хук с `setNow(new Date())` на `setNow(getAdjustedDate())`.
3. Убедиться, что в функцию `getEffectiveTimerState(timerState, blindLevels, now)` передается именно скорректированное значение `now`.

### Шаг 3. Тестирование и верификация
1. Изменить вручную системное время на тестовом компьютере (перевести часы на 5 минут вперед/назад).
2. Открыть экран таймера блайндов `/screen/...` и убедиться, что:
   * Таймер показывает верное оставшееся время раунда.
   * Отсчет идет плавно каждую секунду без скачков при обновлении страницы.
